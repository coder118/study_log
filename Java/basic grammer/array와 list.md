
# 배열

    int[] arr = new int[5]


arr에 5라는 크기의 값이 담기는 것이 아니다. arr은 참조 변수로 5크기의 사이즈를 가진 주소를 가지고 있다.

1. 값을 꺼내거나 설정해줄때 '인덱스'를 사용한다.

2. 여러개의 데이터를 한번에 다룰 수 있다.

3. array는 Object는 아니지만 reference value취급(주소를 참조하는 것)

4. **메모리상 연달아** 공간을 확보하며 **미리 공간을 확보**해서 사용해야 한다. 단, 한번 만들어진 공간은 **크기가 고정**된다.

5. 첫번째 index값만 알면 빠르게 찾을 수 있다. 

==> 단점으로 유연하지 못하다는 문제가 있었다. 사람들은 필요에 따라 크기가 늘어나거나 줄어들면 좋을 것 같은 것을 찾았다.. 그래서 나온 것이 리스트이다. 



### String이 저장되는 방식 

    String[] arr = new String[3]{"a","b","c"}

문자열의 경우 배열 내부에 문자열 값이 들어가는 것이 아니다. 문자열을 가르키는 주소값이 배열에 들어가게 되고 문자열은 heap에 저장되게 된다.

> tip 알면 좋을 정보
Heap 영역에 (String Constant Pool)
실제 "a", "b", "c"라는 문자열 데이터는 Heap 내의 String Constant Pool이라는 특수한 영역에 객체로 생성된다.
배열의 각 요소는 바로 이 Pool에 있는 객체들을 가리키는 것.


# 리스트

리스트가 몇개가 필요한지 모르니 일단 만들어두는 것이다. 한 메모리의 주소만을 저장을 해두는 것이다.

데이터를 추가할때마다 메모리상 어딘가에 존재만 한다면 링크로 연결을 해서 계속 늘릴 수 있다. (linkedList)

1. 배열과 같이 여러개의 데이터를 다룰 수 있다.

2. 메모리 상 연속되지 않아도 된다.

3. 미리 공간 확보할 필요 없다.

4. 필요에 따라 데이터가 늘어나거나 줄어든다.

5. 첫번째 위치로 부터 index로 목표위치를 알려면 한칸한칸 이동해야 함




List 인터페이스 내에 arraylist, vector,linkedlist 다양하다. 하지만 그 각각의 기능이 미묘하게 다르다. arraylist 와 vector는 유사해보이지만 멀티스레드환경에서는 데이터의 정합성을 위해 synch되는 vector를 사용하는게 좋다. 

어떤 때 어떤 것을 사용해야 하냐 
예를 들어, 데이터가 늘어나고 줄어드는 환경이다. 그럼 3가지중 아무거나 선택. 
그 환경에서 인덱스로 빠르게값을 찾아야 한다고 하면 arraylist나 vector중에 한개
synch가 필요하다면 vector이런식으로 선택할 수 있다.
