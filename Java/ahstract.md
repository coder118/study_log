데이터 추상화 (객체의 관련 속성만 '표시')
abstract 를 사용. 
예> 전자제품-> 통신기기->휴대폰-> 아이폰 이런식이다. 전자제품에만 사용되는 기능. 그리고 통신기기에만 사용되는 기능... 이렇게 쭉쭉 늘려가서 상속 받는다. 그럼 자연스레 가장 밑에 있는 아이폰 클래스는 상위 클래스의 모든 내용을 상속받아 사용할 수 있다. 


제어 추상화 - (불필요한 내용을 숨김)
예> 소비자는 자동차의 동작원리가 어떻게 되던 알 필요가 없다. 그냥 핸들, 액셀, 브레이크만 알면된다. 
이런 개념이 코드에 접목이 되는 거다. 객체를 만들어서 사용하는 명령어들 그것이 추상화이다.
