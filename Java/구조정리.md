# 상속(inheritance)

부모 클래스(상위)와 자식클래스(하위)로 나뉘며 하위 클래스는 상위 클래스를 선택해 상위의 멤버를 상속 받아 사용이 가능해진다. 

### 왜 상속하는가?

미리 만들어둔 클래스를 재사용 함으로 효율적이고 개발 시간 감소.

일반적인 클래스가 이미 구현이 되어있는 상태에서 그 보다 좀 더 구체적인 클래스를 구현하기 위해 사용되는 기법이다. 그로인해 상위 클래스의 코드를 하위클래스가 사용할 수 있을 뿐이다.

### 객체가 생성되는 순서?


상위 클래스에서만 선언된 함수를 하위 클래스에서도 그대로 사용할 수 있다. 하지만 클래스를 선언할때 하위 클래스를 먼저 호출했는데 어떻게 상위클래스를 인식하고 함수를 사용할 수 있을까?

그 이유는 객체의 생성 순서이다. 하위 객체를 생성하면 상위 객체를 **먼저 생성한 후** 자식 객체가 생성되는 것.

코드에서 구현할 수도 있고 안 할 수도 있다. 안 할 경우에는 컴파일러가 알아서 기본 생성자를 생성해서 호출한다.
코드로 구현할 경우에는 **하위 클래스의 생성자** 내부 첫줄에 super(매개변수,...)를 선언해주면 된다.

### 상속 사용시 주의점

상위 개념이 하위 개념을 포함할때만 상속을 사용해야 한다.

막 사용하다가 클래스의 결합도가 높아질 수 있다. 또한 상속은 상위 클래스의 구성 자체를 받아오는 것이라서 유연하지 못하다.
이런 문제 때문에 상속보다는 구성(composition)을 사용하라는 것이다.

각 객체마다 할 수 있는 것이 다르고 필요한 기능만 뽑아서 유연하게 사용이 가능하다.

### composition

클래스 필드에 다른 클래스의 인스턴스를 참조하게 만드는 설계이다. 
예를 들어, 한 크래스가 다른 클래스의 기능을 사용하여 구현해야 한다면 composition방식을 사용한다고 보면됨.
학생이 수강하는 과목들, 자동차와 엔진 종류 같은 것.


상속을 사용하지 말라는 소리가 아니다. 
상속은 계층 구조 표현에서 중요하다/ 다형성에서 중요하다. 단순히 중복제거를 위해 상속을 사용하지 말란 이야기다.
-코드 중복제거는 상속보다는 구성(composition)이 유연하다.

# 다형성(polymorphism)

같은 자료형에 여러가지 타입의 데이터를 대입하여 다양한 결과를 얻어 낼 수 잇는 성질이다. 

=> 상위 클래스가 하위 클래스를 같은 메시지로 다른 동작을 하게 만드는 것.

대표적인 활용 예시로 오버로딩, 오버라이딩, 업 캐스팅, 다운 캐스팅, 인터페이스, 추상메서드, 추상클래스 같은 것이다.


### 왜 다형성을 사용해야 하는가?

- 유지보수: 여러 객체를 하나의 타입으로 관리가 가능해 유지보수 용이
- 재사용성: 객체의 재사용이 쉬워 재사용성이 높아짐.
- 느슨한 결합: 클래스간의 의존성을 줄여 확장성은 높아지고 결합도는 낮아진다. 

## 다형성을 구분한다.
### 상속에서의 다형성

부모-자식(상위-하위)와 같이 상속 관계에 있는 클래스라면 부모 타입의 참조변수로 자식 객체를 다룰 수 있다. 
이 말은, **부모 클래스의 참조 변수**로 자식 클래스를 다루면 부모 클래스에 있는 멤버와 메서드를 사용할 수 있다는 것이다.

단, 자식의 참조변수로 부모 객체를 생성하는 것은 컴파일 에러 발생.

### 자료형 다형성

자료형으로 다형성을 활용할 수 있다. 

### 매개변수 다형성

다형성의 특성은 변수 타입 뿐만 아니라 인터페이스나 파라미터에서도 똑같이 적용된다.


### 메서드 다형성

오버 로딩과 오버 라이딩

- 오버로딩(overloading): **같은 이름**의 메서드 여러 개 가지면서 **매개변수의 유형과 개수를 다르게** 하는 것. 컴파일 시 일어나는 정적 다형성이다. 
- 오버라이딩 (overriding): 상위 클래스가 가지고 있는 메서드를 **하위 클래스가 재정의해서 사용**하는 것을 말함. 런타임시 일어나는 동적 다형성이다.

# 추상클래스(abstract class)

추상메서드를 하나라도 가지고 있는 클래스가 추상 클래스가 된다. 추상 메서드는 다른 클래스가 상속 받을때 강제적으로 추상 메서드를 오버라이딩하게끔 만드는 역할이다.

추상클래스는 new, 즉 객체를 생성할 수 없다.
