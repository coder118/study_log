### 일단 내가 이해한 방식

빌더 내부에는 만들어질 제품들의 공통 내용들이 있다.

예를 들어 빌더라는 인터페이스가 있고 자동차빌더클래스와 자동차 매뉴얼빌더라는 클래스(=concrete builder)가 있다. 그럼 이 두가지의 공통되는 내용이 빌더 인터페이스에 존재한다.
오리지널 자동차 클래스와 오리지널 자동차 매뉴얼 클래스는 따로 존재한다. 위에서 빌더는 이 오리지널을 만드는 클래스들의 정보를 모아서 오리지널로 값을 보내주는 거다.

클래스들의 정보를 모은다고했는데 기능별로 클래스 묶음을 만든다. 자동차에 들어가는 기능들을 클래스로 분리시켜놓은 거야. 엔진, 네비게이션, 전조등에 대한 정보 각각의 클래스를 만들어서 분리해놓았다.

디렉터는 뭐냐? 조립 순서를 캡슐화 시킨 것이다. 동일한 제작 절차로 서로 다른 product를 만든다. 복잡한 순서를 클라이언트에서 제거한 것이다.
