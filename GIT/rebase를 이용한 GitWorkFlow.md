# 깃 워크플로우 정리 (협업)
1. 일단 내가 만들 이슈(issue)를 깃허브에 올린다.

2. 이슈의 번호를 포함해서 브랜치를 만든다. 브랜치를 만들때 가장 최신의 origin/main상태면 좋다.

3. git branch <브랜치명> 을 짜서 만든다(git branch -m으로 브랜치명 수정 가능)

4. 해당 브랜치에서 개발을 진행한다. 나의 브랜치이기 때문에 자유롭게 활동할 수있다.

5. 개발이 완료되면 바로 push하면 안된다.

6. 다른 팀원이 origin/main에 이미 머지를 해놓았을 수 도 있기 때문에 나는 내 브랜치에서 git pull origin main(무조건 main이 아니니 주의요망!!! 팀별로 사용하고 있는 브랜치가 다를 위험이있다.) --rebase를 실행한다. (해당 명령어는 git fetch origin과 git rebase origin/main을 줄여놓은 것이다.)

    6-1. pull이 안되는 경우가 있다. 아마 변경사항이 스테이징에 올라갔지만 커밋되지 않은 상황일 것이다.

    6-2. 해결책은 2가지다. git add . , git commit -m"작업중" 을 해서 작업한 내용을 커밋으로 만든다.
    2번쨰 방법은 git stash를 사용하는 것이다. 변경사항을 다른 곳에 임시로 저장해두고 git pull ....을 실행하고 git stash pop을 하면 된다. (주의점. pull한 내용이 나의 stash와 겹쳐서 충돌이 발생할 수 있다. 하지만 내가 작업한 부분이 다른 사람이 개입하지 않는다면 충돌나는 일은 없을 것이다)

7. 가장 최신 origin/main의 가장 최근 커밋 내용을 나의 브랜치가 가르키게 된다. 한마디로 origin/main보다 앞서 있게 된다. (이렇게 되면 팀원들의 변경사항을 내 브랜치에서 확인해서 적용하는 것이 가능하다.)

8. 이제 git push를 하려고하면 --set -upstream이 뜨면서 내브랜치를 원격에 올리라고 뜰것이다. 그 과정을 그대로 진행한 뒤 원격에서 확인해보면 내 브랜치와 커밋 내역이 잘 올라가 있는 것을 확인할 수 있다.

    8-1. rebase하면 커밋의 해쉬정보가 바뀐다. 예를 들어, 이미 push된 값이 있는 브랜치에서 작업을 진행하고 새로운 커밋을 만들었다. 마찬가지로 pull....rebase를 실행하고 push하려고 하면 내 원격에 올라가있던 기존 커밋 해쉬와 지금 pull하면서 변경된 커밋 해쉬가 일치하지 않아서 push가 되지 않을 경우가 있다.

    8-2.git push --force를 사용하는 방법도 있지만 이건 정말 강하게 올리는 것이라서 만약 다른 팀원의 작업 내용을 다 날려버릴 수 있다. 그래서 git push --force-with-lease를 사용하면 좋다. 이렇게 되면 다른 팀원이 이미 커밋된 상황이라면 강제 push를 멈춰줘서 협업때 많이 사용된다고 한다.

9. 이제 나의 로컬로 돌아와서 로컬 main이 뒤쳐져있다면 git pull 명령어를 사용해서 origin/main이 가르키는 최신 커밋 내용을 가져온다.

10. 최신화된 내 로컬 main에서 새로운 이슈 브랜치를 생성해서 작업을 할 수 있다.

# PR할때

1. 원격에 올라간 내 브랜치를 main으로 PR을 날린다.

2. 사람들이 코드 리뷰를 해줄 것이다. 어떻게 바꾸면 좋을지 이야기를 해주면 난 다시 내 로컬 브랜치로 돌아와서 작업을 수정하고 커밋 메시지에 fix라는 내용을 포함해서 커밋을 남긴다.

    2-1. 수정하는 사이에 또 변경되었을 수 있으니까 git pull....rebase를 해주고 push 한다.

3. 아마(?) 다시 PR을 날릴 수 있다. 다시 PR을 날린다. => 로컬에서 수정 후 push만 하면, 이미 올라가 있는 PR에 자동으로 커밋이 추가되어 업데이트됩니다. 리뷰어는 그냥 새로고침만 하면 바뀐 코드를 볼 수 있다고 함.

4. origin/main에 merge하는 것에 성공했다.

5. 그럼 깃허브에 해당 브랜치를 삭제할 것이냐고 뜬다. 그것을 삭제하고 로컬로 돌아온다.

6. 원격에서 삭제된 브랜치가 내 로컬에는 그대로 존재한다. 그래서 git remote prune origin 하면 원격 브랜치를 삭제해 줄 것이다.

7. 그리고 로컬 main으로 이동해서 내가 origin/main에 머지한 내용을 pull 해서 가져온다.

8. 그리고 내가 작업하던 로컬 브랜치를 삭제하면 될 것이다
