# 깃 내용 정리
---
Git Flow 라는 개념을 이해하는 중이다. 

git에서 버전이란 문서를 수정하고 저장할 때마다 생기는 것이라고 생각하며 간단하다.
예를 들어 '수정'이라는 파일이 있고 그 파일이 1000개가 넘는다고 할때, 그걸 다른 이름으로 파일들을 저장할 수 있을까?

깃에서 버전을 관리하면 원래 파일이름은 그대로 유지하며 파일에서 무엇을 변경했는지를 변경시점마다 저장 할 수 있다. 또 각 버전마다 작업했던 내용을 확인할 수 있고 그 버전으로 돌아갈 수도 있다. 

---
## 깃이 동작하는 원리

작업 트리(working tree)-> 스테이지 ->저장소의 형태로 이루어짐

### 작업 트리 = working directory 

파일 수정,저장, 등의 작업을 하는 디렉토리

### 스테이지

버전으로 만들 파일이 대기하는 곳
예> 작업트리에서 10개의 파일을 수정했는데 4개의 파일만 버전으로 만들려면 4개의 파일만 스테이지로 넘기면 됨

### 저장소 

스테이지에서 대기하고 있던 파일을 버전으로 만들어 저장하는 곳이다. 

# git 명령어

    git status //깃의 상태를 나타내는 명령어
    git add <file이름> //파일을 스테이지로 올림
    git commit -m "입력 내용" //스테이지에 올린 파일을 저장소로 옮김
    git commit -am "입력 내용" //한 번 커밋된 파일일 경우 스테이지와 커밋을 동시에 진행하는 옵션 
    git log //커밋 내용 살펴보기
    git diff //방금 수정한 파일을 저장소의 최신버전과 비교
--- 
    git branch <브랜치이름>//깃 브랜치를 확인하고 만듬
    git checkout <브랜치이름> //다른 브랜치로 이동
    git log --oneline //한줄에 간편히 보기
    git log --oneline --branches --graph //브랜치 시각화
    git merge <병합할 브랜치 이름>
    git stash //커밋하지 않고 작업 중인 파일들을 잠시 감춰둘 수 있음
        -pop //stash 목록에서 가장 최근항목을 되돌림
        -apply //가장 최근 항목을 되돌리지만 저장한 내용 그대로 남겨둠.
