# AWS란?

amazon web service이다.
<br>
전 세계적으로 분포한 데이터 센터에서 다양한 기능을 제공해주는 클라우드 플랫폼이다.

## 클라우드 컴퓨팅?

- IT 리소스를 인터넷을 통해 on Demand로 제동하고 사용한 만큼만 비용을 지불하는 방식이다.

- 물리적 데이터 센터와 서버를 구입, 소유, 유지 관리하는 대신, AWS 같은 클라우드 공급자로부터 필요에 따라 컴퓨팅 파워, 스토리지, DB와 같은 기술 서비스에 엑세스 한다.

> 클라우드 컴퓨팅 장점

- 민첩성 
- 탄력성 : 필요없는 서버가 생기면 잠시 내려두고 부족한 리소스가 있으면 추가하고 혹은 제거가 간단하다
- 비용절감 
- On Demand
- 관리 용이 : 서버실을 들어가서 관리하는 것이 아니다. 우리는 GUI,CLI를 사용해서 클라우드 플랫폼이 제공해주는 것을 사용하면 됨

## 클라우드 유형

![클라우드 유형](/AWS/aws_images/aws.png)

- IaaS(Infrastructure as a Service): 클라우드를 통해 컴퓨팅, 스토리지, 네트워킹, 가상화와 같은 주문형 인프라 리소스를 조직에 제공 => 하드웨어를 대여하여 애플리케이션을 실행하지만 OS, 런타임, 확장 및 모든 데이터를 관리할 책임은 본인
- PaaS(Platform as a Service) :  클라우드를 통해 애플리케이션을 개발하는 데 필요한 모든 하드웨어 및 소프트웨어 리소스를 제공하고 관리 => 자신의 코드를 가져와서 배치하지만 서버 관리와 확장은 클라우드 제공업체에 맡길 수 있다.
- Software as a service(SaaS) :  전체 애플리케이션 스택, 즉 고객이 액세스하고 사용할 수 있는 전체적인 클라우드 기반 애플리케이션을 제공 => 클라우드 제공업체가 관리, 유지보수, 보호하는 완전한 애플리케이션을 특정 목적으로 사용하기 위해 비용을 지불, 하지만 자신의 데이터 관리는 자신이 책임

다양한 클라우드 제품들 : aws, azure, gcp, oci, ncp 등..

# AWS 기본 용어

### 가상화

물리적 **컴퓨터 하드웨어를 보다 효율적으로 활용**할 수 있도록 해주는 프로세스이며, 이는 클라우드 컴퓨팅의 기반을 제공하는 기술

### 가상머신

소프트웨어 형식으로 물리적 컴퓨팅을 시뮬레이션하는 가상환경이다. 가상화한 것을 논리적으로 연결해서 사용.


### 스냅샷

**특정 시점**에 스토리지 파일 시스템을 포착해서 보관하는 기술.

- 장애, 데이터 손상시 스냅샷을 생성한 시점으로 데이터 복구
- 백업과 다르게, 초기 생성시 혹은 데이터 변경이 있기 전까지 스토리지의 공간을 차지하지 않는다.
- 메타 데이터의 복사본에 해당되기에 생성시간이 짧고, 빠른 복구가 가능

### 데이터 센터

수많은 서버를 한데 모아 네트워크로 연결해놓은 시설

### 지역(region)

데이터 센터가 위치하느 지역이며, 대상 고객읭 지역과 자원 생성할 region이 최대한 가까워야 한다.

### Availability zone(가용영역)

하나의 region은 두개 이상의 availability zone으로 구성된다.

- 줄여서 **AZ 로 표시한다**.

# EC2 기능 정리

![ec2구성도](/AWS/aws_images/aws2.png)

- 인스턴스 : 가상 컴퓨팅 환경
- AMI(amazon 머신 이미지) : 서버에 필요한 Os와 여러 sw들이 적절히 구성된 상태로 제공되는 템플릿으로 인스턴스를 쉽게 만들 수 있다. 
- 인스턴스 유형 : 인스턴스를 위한 CPU, 메모리, 스토리지, 네트워킹 용량의 여러가지 구성 제공
- 키 페어 :만든 인스턴스 로그인 정보 보호(aws는 public key를 저장하고 사용자는 private key를 안전한 장소에 보관)
- 인스턴스 스토어 볼륨 : 임시 데이터를 저장하는 스토리지 볼륨, 인스턴스 중단, 절전모드 전환..
- amazon EBS: EBS 볼륨을 사용해 영구 스토리지 볼륨에 데이터 저장 = > 인스턴스의 볼륨을 설정
- 보안 그룹 : 인스턴스에 연결할 수 있는 프로토콜, 포트, 소스 IP 범위를 지정하는 방화벽 기능
- 탄력적 IP 주소(EIP) : 동적 클라우드 컴퓨팅을 위한 고정 ip v4 주소
- 태그 : 사용자가 EC2리소스에 할당할 수 있는 메타 데이터
- VPC(virtual private clouds) : aws 클라우드에서 논리적으로 격리되어있지만 원할때 마다 고객의 네트워크와 간편히 연결할 수 있는 가상 네트워크
