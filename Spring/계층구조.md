# 계층형 구조

### DTO (Data transfer object)

계층 간 데이터 교환을 하기 해 사용되는 객체
어떠한 로직도 가지지 않고 속성과 그 속성에 접근하기 위한 getter,setter 메소드를 가진 클래스

### Entity

DB 테이블에 매핑되는 객체이다. JPA를 사용하는 애플리케이션에서는 각 Entity가 db테이블의 한 행과 대응된다.

### controller

사용자의 요청을 받아 처리.
@autowired로 Service를 주입 받아 비즈니스 로직의 결과를 사용자에게 전달함.

요청 수신
요청 검증
서비스 호출
응답 반환

### service

비즈니스 로직을 수행함. @autowired로 DAO를 주입함.
여러 DAO를 조합해 복잡한 비즈니스 요구사항 충족
또한 필요한 데이터를 DTO 또는 도메인 객체 형태로 Controller에게 전달한다.

### DAO(Data Access Object)=Repository와 유사

DB의 데이터에 접근하기 위한 객체. 실제 DB에 접근.
JPA에서는 DB에 데이터를 CRUD하는 Repository객체들이 DAO이다.
DAO는 DB와 데이터에 관련된 로직을 처리한다.

일반적으로 '인터페이스 - 구현체 ' 구성으로 생성.

### repository

JPA의 JpaRepository를 상속받아 구현된다.
