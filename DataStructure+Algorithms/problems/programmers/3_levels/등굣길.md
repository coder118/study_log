# 문제 정의 및 처음 풀이

- 오른쪽 이동 -> x좌표가 증가
- 아래 이동 -> y좌표 증가

오른쪽 아래쪽으로 이동을해서 (m,n)좌표 위치에 도달할 수 있는 최소 경로 수를 구해라.

# 처음 생각했던 풀이

- visited 처리를 해야 한다고 생각했는데=> 갔던 길을 또 갈 수 있기 때문에 visited를 통해 갔던 길을 막는 것은 안된다.
- 그럼 갔던 길을 기록해야된다.=> 변경되는 좌표값을 StringBuilder를 사용하여 문자열을 하나 만들고 그 값을 리스트에 넣어둔다. 
    - 이후, 경로를 지나가려할때, 리스트 내부에 있는 길인지 아닌지를 찾아본다. 
    - 백트래킹? 같은 느낌으로 경로가 진행되었을때 만들어지는 경로가 리스트 내부에 있으면 다시 이전 좌표로 돌아와서 다른 좌표로 이동.

이렇게 되면 복잡도가 증가하게 되고, 풀이의 접근이 잘 못되었다는 생각이 들었다.

# 이후 풀이

- 각 좌표 위치에서 이동할 수 있는 경로를 기록하는 것이다. 

![문제 풀이 이미지](/DataStructure+Algorithms/problems/programmers/images/등굣길.png)

위의 이미지를 기반으로 코드를 작성했다. 

    class Solution {
        public int solution(int m, int n, int[][] puddles) {
            int[][] dp = new int[n + 1][m + 1];
            int mod = 1_000_000_007;

            for(int[] temp : puddles){
                dp[temp[1]][temp[0]] = -1;
            }

            dp[1][1] = 1;
            for (int i = 1; i < n + 1; i++) {
                for (int j = 1; j < m + 1; j++) {
                    if(dp[i][j] == -1){
                        continue;
                    }
                    if(dp[i - 1][j] > 0){
                        dp[i][j]  = (dp[i][j] + dp[i - 1][j]) % mod;
                    }
                    if(dp[i][j - 1] > 0){
                        dp[i][j] = (dp[i][j] + dp[i][j - 1]) % mod;
                    }
                }
            }

            return dp[n][m];
        }
    }

dp 의 크기를 m,n 보다 한개씩 늘린다. 0으로 시작하는 인덱스를 깔끔하게 1부터 시작하게 만들기 위해서이다. 그래서 1,1위치를 1로 초기화하여 시작해서 값을 탐색한다. 
<br>
오른쪽 이동은 dp의 행값이 바뀌어야 하고 아래쪽 이동은 dp의 열값 인덱스가 바뀌어야 한다. 그리고 그 위치에 있는 값이 0보다 커야지 이동할 수 있는 길이 있다는 것이 된다.

최종적으로 (m,n)에 도착하면 도달하기까지 필요한 경로들의 합이 주어지기에 정답을 구할 수 있다.
