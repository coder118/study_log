# 문제 해결 과정

문제에서 주어진 lines(2차원 배열)는 범위를 주고 있다고 생각했다. 그래서 IntStream을 이용해서 범위에 해당하는 set을 만들고 겹치는 값을 제거하면 될 것 이라고 생각했다.

1. 동일한 값 2개= 1칸 , 동일한 값 3개면 =2칸.. => (0,1,2) (0,1,2,3)여기서 0,1과 동일하고 겹치는 선분은 1개가된다.
2. 이 겹치는 영역을 계산하기 위해서 교집합이라는 개념이 떠올랐다.
3. lines 내부(1차원)에 주어진 범위를 set으로 만든다.

# 코드


    Set<Integer>[] rangeSet = new HashSet[lines.length];

    // 배열 초기화
    for (int i = 0; i < rangeSet.length; i++) {
        rangeSet[i] = new HashSet<>(); // 각 요소에 HashSet 할당
    }

    for(int i = 0; i<lines.length; i++){

        int start = lines[i][0];
        int end = lines[i][1];
        //범위 만큼 int set을 생성
        rangeSet[i].addAll(IntStream.rangeClosed(start,end)
                .boxed()
                .collect(Collectors.toSet()));
    }

    Set<Integer> intersection = new HashSet<>();
    
    // 구한 set에서 겹치는 값을 찾는다.
    for (int i = 0; i < rangeSet.length; i++) {

        for(int i2 = 1; i<rangeSet.length; i++){
            intersection = rangeSet[i]; // 첫 번째 Set을 복사
            intersection.retainAll(rangeSet[i2]);

            if(!intersection.isEmpty()){
                answer += intersection.size()-2;
            }
        }
    }

겹치는 값을 찾는 과정에서 문제가 발생했다. 

1. 여러 가지의 숫자 범위가 겹쳤을때 계산 처리하는 것이 너무 복잡해진다.
2. lines 내부에 3개의 값이 있다고 했을때 (0,1)/(0,2)/(1,2)를 각각 처리해서 겹치는 값을 찾아야 한다.

지금까지 적은 내용도 충분히 오버엔지니어링인데 여기서 로직을 더하면 가독성도 떨어지고 복잡도가 늘어나는 코드로 변할 것이라고 판단하여 폐기하였다.

# 해결책

겹치는 구간이 (0,3)이라고 했을때 0,1,2,3이렇게 값이 4개 존재한다고 선분의 길이가 4가 아니라  실제 선분의 길이는 3이다.
<br>
다른 구간도 계산해보면 마찬가지다. 총 size에서 1을 뺀 것과 동일하다.
<br>
그렇다면, 겹치는 점이 있을때마다 늘려주고 마지막에 겹쳤던 점이 몇개인지 카운팅하면 선분의길이가 나오지 않을까?

    Map<Integer,Integer> save= new HashMap<>();
            
    for(int i =0; i<lines.length; i++){
        for(int i2 = lines[i][0]; i2<lines[i][1]; i2++){
            save.put(i2,save.getOrDefault(i2,0)+1);
        }
    }

    for(Map.Entry<Integer,Integer> entry : save.entrySet()){
        if(entry.getValue()>1){
            answer++;
        }
    }

맵으로 겹치는 값이 있으면 value값을 늘려주는 방식이다. 만약 값이 존재하지 않는다면 0을 반환한다.
