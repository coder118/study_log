# 문제 정의

    1 ≤ b < a ≤ n ≤ 1,000,000

제한 사항을 보았을때 n^2 시간 복잡도를 사용하면 시간 제한 초과가 일어난다. 이중 반복문은 배제하고 문제를 풀었다.

> **문제 정리**

1. a개 빈병을 가져다주면 콜라 b개를 준다.
2. n개의 빈병이 있다.
3. 내가 총 받을 수 있는 b의 개수는 몇개인가?

> 문제 정의

1. n/a = (몫*b) 콜라를 받는다.
2. (몫*b)를 다 먹고 빈병이 된다. n = (몫*b) + 바꾸지 못한 병
3. n/a = (몫*b) 콜라를 받는다.
4. .... 쭉 반복되다가 n의 개수가 a 보다 작아지면 콜라를 받을 수 없다.
5. 지금껏 받았던 총 b는 어딘가에 저장되어있다가 합을 구하면된다.

## 해결 방법 고민

나눈 몫이 전체 개수가 되고 그 전체 개수는 다시 a로 나뉘는 과정을 보고 재귀함수가 떠올랐고 하나씩 구현을 시작했다.

# 코드 

    class Solution {
        
        public static List<Integer> save = new ArrayList<>();
        public int solution(int a, int b, int n) {
            int answer = 0;
    
            check(n,a,b);
            
            for(int i=0; i<save.size(); i++){
                answer+= save.get(i);
            }
            
            return answer;
        }
        public void check(int total, int base, int get){
            
            if(total<base){
                return ;
            }
            
            int receieved=(total/base)*get;
            save.add(receieved);
            
            total = receieved + (total%base); //바꾸지 못한 남은 병을 더한다.
            
            check(total,base,get);
        }
    }

미리 생각하지 못한 점은 남은 병을 처리하는 것을 잊고 있었다. 그래서 전체 개수에서 base를 나누어서 나오는 나머지 개수를 받은 병값(receieved)과 더해주었다.

# 피드백

굳이 save라는 외부 전역변수를 사용할 필요가 있었나 싶다. 다른 사람 코드에 내가 구현하고 싶었던 방식이 있었다.


    class Solution {
        public int solution(int a, int b, int n) {
            return recursive(a, b, n, 0);
        }

        public int recursive(int a, int b, int n, int result) {
            if (n < a) {
                return result;
            }

            int change = (n / a) * b;

            n = (n % a) + change;

            return recursive(a, b, n, result + change);
        }
    }

마지막에 변수하나를 더 추가해서 change와 result값을 계속 더해가면서 받아서 먹는 콜라의 개수를 세면서 마지막에 조건이 끝날때 result를 반환해서 바로 결과값으로 받는다.
