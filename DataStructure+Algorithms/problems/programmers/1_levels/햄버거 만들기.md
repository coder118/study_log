# 문제 정의

원래는 배열을 반복문 돌려서 하나씩 확인하는 것을 생각했지만 1231이 발견되면 배열 내부의 값이 삭제되어야 하기때문에 단순한 반복문은 아닐 것으로 생각했다.

- 이전의 값을 기억 해야 하는것?
- 1231이라는 패턴을 가지게 하는 방법

패턴을 만들어서 기억하고 제거하는 방법을 실행하려면 문자열이 적합할 것 같다는 생각이 들었다.


## 문제 풀이

    StringBuilder sb = new StringBuilder();
    for(int i: ingredient){
        sb.append(i);
    }

    String answer = sb.toString();
    int count =0;
    boolean check = true;
    while(check){
        if(answer.contains("1231")){
            answer = answer.replaceAll("1231","");
            count++;
        }else{
            check = false;
        }
        
    }

1231이 나올때마다 공백으로 replaceAll을 사용하는 방법을 택했다. 하지만 불안했던게 String은 호출할때마다 새로운 메모리에 생성된다는 점이었다. StringBuilder에는 replaceAll이없어서 사용하지 못했다. 결과는 좀 성공하다가 예상했던 문제로 실패했다.

## 다른 해결책

String이 아니라 StringBuilder을 사용하면 해결 할 수 있을 것이라는 생각이 들었다. 

<br>

문제되었던 부분은 한번에 sb에 값을 다 넣어버리면 1231을 비교할 수 없고 만약, 1231을 지운다하더라도 반복문을 새로 돌려야 한다는 문제점이 있었다.

=> 반복문내에서 쌓으면서 비교를 하는 방식을 선택했다.

   
    int count = 0;
    StringBuilder sb = new StringBuilder();
    for (int i : ingredient) {
        sb.append(i); // 일단 재료를 쌓는다.

        // 재료가 4개 이상 쌓였을 때만 체크
        if (sb.length() >= 4) {
            // 마지막 4개 글자가 "1231"인지 확인
            if (sb.charAt(sb.length() - 4) == '1' &&
                sb.charAt(sb.length() - 3) == '2' &&
                sb.charAt(sb.length() - 2) == '3' &&
                sb.charAt(sb.length() - 1) == '1') {

                count++;
                // 1231 패턴을 찾았으므로 뒤에서 4개를 삭제 (고속 삭제)
                sb.setLength(sb.length() - 4); 
            }
        }
    }

sb에 값이 4개 쌓였을때 값을 비교한다. 1231로 쌓여있으면 제거하고 아니면 계속 값을 쌓는다.
<br>

여기서 **중요한 부분은 setLength**이다. sb의 길이를 조정해주는 것이다. 
<br>
예를 들어, 1234567값이 있을때 setLength(2)를 실행해준 뒤 append(8)을 해주면 7뒤에 값이 붙는게 아니라 3자리에 8이 들어간다.
<br>
setLength는 만들어둔 메모리를 삭제하지도 않고 그대로 냅두고 append가 가르키는 포인터의 위치를 바꾼다고 생각하면 된다. 
<br>
그래서 겹치는 값 1231을 제거하는 것이 아니라 신경쓰지 않고 위치를 앞으로 보내버리는 것이다. 그리고 새로운 값을 덮어씌우는 방식이다. 

# 배운점

setLength 사용법에 대해 알게 되었고 메모리 문제를 좀 더 신경쓰면서 풀어야 겠다는 생각이다.
