# 우선순위 큐를 사용한 문제들

## 1번 문제 조건

> 문제 정리

1. n은 병사 수, enemy[]는 라운드 마다 적의 마리 수
2. enemy[i]마리 만큼 n명의 병사가 막을 수 있다. -> 2마리 적 = 2명의 병사

3. 남은 병사(n) < 현재 라운드의 (enemy[i]) 적의 수 -> 게임 종료

4. 무적권 k개 존재함. 병사(n)을 줄이지 않고 적(enemy[i])를 막을 수 있다.

5. 최대 라운드를 진행하고 싶다. 

> 문제 풀이 순서도 작성

1. 라운드 순서대로 k를 사용하면 안된다. 
2. enemy[i]의 수가 n보다 클 경우 k가 남아있다면 k를 사용해야 한다.
3. n > enemy -> 무조건 n을 소모해야 하는가?
     기준을 어떻게 세워야 하지?
    1. n을 소모시키는 방법
    2. n을 아끼고 k로 방어후 뒤에서 n을 사용
         => n을 언제 아끼면 좋을까?

위와 같이 생각했다가 접근법을 바꾸었다.

1. enemy가 가장 클때 아껴두었다가 k을 사용하는게 효율적
2. 근데 가장 클때를 알수가 없지. -> 저장을 해두어야 한다. 
3. n이 부족해지고 k가 남아있는 상태에 저장된 값이 존재한다면, 저장한 값에서 가장 큰 값을 n에 복구하고 k를 사용

------

### 1번 문제 코드 구현

    PriorityQueue<Integer> pq = new PriorityQueue<>(Collections.reverseOrder());

    for(int e :enemy){
        n-=e;
        pq.add(e);
        //병사가 부족해지면
        if(n<0){
            
            //k가 존재하고 pq의 값(=지금까지 진행된 라운드에서 가장 많은 적)이 존재할때
            if(k>0 && !pq.isEmpty()){
                n+=pq.poll(); //적이 가장 많을때 사용한 병사를 복구하고
                k--; //k = 무적권 사용
            }else{
                //무적권도 없고 복구할 병사수도 없다면 종료
                break;
            }
        }
        answer++;
    }

## 2번 문제 조건

1. scoville 내부에 있는 배열이 K이상의 값을 가지게 하고 싶다.
2. 규칙ㅡ 섞은 음식의 스코빌 지수 = 가장 맵지 않은 음식의 스코빌 지수 + (두 번째로 맵지 않은 음식의 스코빌 지수 * 2)
3. 가장 최소값 +(2번째로 맵지 않은 음식*2) => scoville 내부에 삽입
4. scovilledms 1,000,000 이하여서 이중 반복은 어렵. 특히 K는 10억이하임.

> 문제 해결 순서도

1. 우선순위 큐의 최소 힙을 활용한다.
2. K보다 작은 값이 나올 경우 계속 반복을 돌면서 스코빌 지수를 높힌다. 2번 규칙을 따른다.
3. scoville 배열은 2이상이 되어야 한다. 만약 2보다 작아지면 break한다.

### 2번 코드 구현

    //최소 힙
    PriorityQueue<Integer> pq = new PriorityQueue<>();

    for(int v : scoville){
        pq.add(v);
    }

    // 최소 힙이므로, peek값에는 K보다 작은 값이 나올 것 = 섞어서 스코빌 지수를 높여줘야 함.
    while(pq.peek()<K){
        
        if(pq.size()<2){
            //scoville의 길이는 2 이하여야 한다. 
            return -1;
        }

        int first = pq.poll();
        int sec = pq.poll();

        // 스코빌 지수를 높힌다.
        pq.add(first +(sec*2));

        answer++;
    }
