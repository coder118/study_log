
# 문제 정의

- edges 노드의 정수 번호 
- 배열 내부는 [현 노드, 자식 노드]로 구성되어있다. 
- s번 노드에서 너비 우선 탐색(bfs) 시작

1. 배열 내부에 s값이 존재하는지 찾는다.
2. 현 노드의 자식 노드가 존재하면 그 값을 따로 저장하고 현 노드는 방문 처리한다. 

queue, set, List를 사용하면 방문처리와 저장을 처리해줄 수 있다.
<br>
queue는 노드의 정보를 저장하고 set는 중복값을 내부적으로 처리해주기 때문에 만약 방문한 노드라면 알아서 처리된다.

<br>
코드로 설명을 해보자면 아래와 같다.

## 코드

    import java.util.*;
    class Solution {
        
        public ArrayList<Integer> solution(int[][] edges, int s) {
            HashMap<Integer,ArrayList<Integer>> graph = new HashMap<>();
            
            for(int i =1; i<=100000; i++){
                graph.put(i,new ArrayList<Integer>());
            }
            
            for(int i = 0; i<edges.length; i++){{
                graph.get(edges[i][0]).add(edges[i][1]);
                graph.get(edges[i][1]).add(edges[i][0]);
            }}
            
            for(int i =1; i<=100000; i++){
                Collections.sort(graph.get(i));
            }
            return bfs(graph,s);
        }
        
        ArrayList<Integer> bfs(HashMap<Integer,ArrayList<Integer>> graph,int start){
            ArrayList<Integer> path = new ArrayList<>();
            HashSet<Integer> check = new HashSet<>();
            Queue<Integer> que = new LinkedList<>();
            que.add(start);
            check.add(start);
            
            while(!que.isEmpty()){
                Integer cur  = que.poll();
                path.add(cur);
                
                for(Integer n : graph.get(cur)){
                    if(!check.contains(n)){
                        check.add(n);
                        que.add(n);
                    }
                }
            }
            return path;
        
        }
    }

graph에 100000이라는 값을 넣어준 이유는 문제에서 제시된 최대값이었다. 100000이라는 값은 컴퓨터에서 그렇게 부담스런 값이 아니기 때문에 s값부터 bfs를 실행해서 값을 편하게 찾기 위해서 세팅을 해둔 것이다.

<br>
bfs 함수 로직은 queue에는 현재 노드의 값을 입력한다. queue에는 bfs를 하면서 찾는 노드의 값을 저장하고 queue내부의 값이 비어있을때 반복문이 중지된다. 노드를 찾을때 방문한 노드인지 아닌지를 확인하고 방문하지 않은 노드일때 방문처리하고 queue에 값을 저장해준다. 
<br>
마지막으로, bfs한 결과를 path 넣어서 반환해주는 것이다.
