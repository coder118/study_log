# 나의 사고 과정

값이 저장될 스택을 하나 만들어서 기존에 있는 값과 새로 들어오는 값이 동일할 경우 pop()함수를 사용해서 stack을 비운다.

1. moves(크레인 동작)배열을 반복해서 인형을 뽑는다.
2. board의 위치값은 moves[]-1 이다.
3. 단순히 2차원 배열 형태로 값을 출력하면 안된다. 
(0,0)/(1,0)/(2,0)/(3,0)... 이렇게가 1번 열이다.
(0,1)/(1,1)/(2,1)/(3,1)/... 이렇게 2번 열이다.
4. 크레인으로 잡은 값을 stack에 넣는다.
5. stack 넣은 값을 0으로 바꿔준다.
6. stack내부의 값이 존재할때 peek()로 출력해 새로 들어오는 값과 비교한다.
    - stack에서 pop()한다.
    - answer값을 2를 증가시킨다. 같은 인형일때 부서지는 인형의 개수는 2개이기 때문이다.

# 시간 복잡도 

이중 for문으로 작성했다. O(N^2)의 복잡도를 가지고 있다.
moves변수의 범위는 1~1000이하이다. moves를 for문으로 돌리면 O(N)의 복잡도를 가지더라도 더 큰 복잡도인 O(N^2)에 묻혀 신경쓰지 않아도 된다.
