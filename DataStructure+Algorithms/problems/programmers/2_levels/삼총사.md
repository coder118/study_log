# 문제 정의

1. 각 학생별로 정수 번호를 가짐
2. 3명의 학생을 더했을때 0이 되면 3명의 학생은 "삼총사"라고 함

> 목표
number 배열에서 3개를 조합해서 0이 되는 경우를 찾아라.

- 인덱스가 0,1,2위치에서 더해서 0이 나올때 0,1,3에서 0이 나올때 이런 경우를 생각해야 한다.

1. count가 3이면 종료가 되어야 하며, 이때 sum이 0이면 answer를 카운트한다.
2. 시작값이 전체 길이보다 길어지면 종료한다. (한번 다 돈 경우를 말함)
3. 반복하는 start가 항상 바뀌어야 한다. 0에서 조합되는 경우 1에서 조합되는 경우


# 코드 풀이

    int[] number;
    int n;
    int answer;

    public int solution(int[] number) {
        this.number = number;
        this.n = number.length;
        this.answer = 0;

        backtrack(0, 0, 0); // start=0, count=0, sum=0
        return answer;
    }

    private void backtrack(int start, int count, int sum) {
        // 3명 선택했을 때 합이 0이면 정답++
        if (count == 3) {
            if (sum == 0) answer++;
            return;
        }
        
        if (start >= n) return;

        for (int i = start; i < n; i++) {
            backtrack(i + 1, count + 1, sum + number[i]);
        }
    }

시작 위치가 계속 바뀌게끔 for 문 내부의 시작값을 start로 만들어주고 backtrack을 계속 호출하면서 인덱스 위치가 0일때의 조합, 1일때의 조합이 만들어진다. 
<br>
이때 만들어지면서 count를 통해 3번 이상 넘어가지 않으며 sum이 0이 된다면 answer을 카운트해서 최종적인 결과값을 구한다.
