# 나의 사고 과정

> 주어진 조건 분석

bridge_length-> 다리길이
weight-> 다리 위에 올라갈 수 있는 무게
truck_weights-> 트럭 별 무게

truck_weights(배열)가 10,000이하 N^2의 시간복잡도를 가지면 된다. 딱 1억번으로 약 1초의 시간이 소요된다.

> 알고리즘 풀이

1. 다리를 큐(bridge_queue)로 만든다. 사이즈는  bridge_length의 크기 만큼이다.
2. truck_weights의 값을 bridge_queue에 넣는다. 
3. bridge_queue의 무게가 weights를 비교해서 초과할 경우 어떤 변경 사항없이 time을 흐르게 한다.
    -무게를 초과하지 않을 경우 기존의 트럭을 앞으로 이동시키고 새로운 트럭을 넣는다.
    - bridge_queue의 무게를 재측정한다.(새로운 트럭이 들어온 값을 더한다.)
4. time의 값을 반환한다.

## 문제 발생

    for(int truck : truck_weights){
        time++;
        bridge_weight += truck;
        if(bridge_weight >weight){
            truck_out_check =bridge_queue.poll();
            if(truck_out_check!=0){ //빠져나가는 값이 0이 아니면 트럭이 나간다는 뜻이다.
                bridge_weight -= truck_out_check;
                truck_out_check =0;
            }
            bridge_queue.offer(0);//뒤에 0을 짚어 넣는다.
        }else{
            truck_out_check =bridge_queue.poll();
            if(truck_out_check!=0){ //빠져나가는 값이 0이 아니면 트럭이 나간다는 뜻이다.
                bridge_weight -= truck_out_check;
                truck_out_check =0;
            }
            bridge_queue.offer(truck);
            
            
        }
        
    }

트럭의 반복이 끝났다고 다리위에 있는 트럭이 다 나간 것이 아니다. 다리 길이만큼 트럭을 반복시켜야 하는 것이 아니다. 

> 추가 정리 조건

truck_weights가 다 이동하고 bridge_weigth가 0이될때까지의 시간을 구해야 한다.

## 해결

    ArrayDeque<Integer> bridge_queue = new ArrayDeque<>();
        
    for(int i = 0; i< bridge_length; i++){
        bridge_queue.offer(0);
    }

    int bridge_weight = 0;
    int time =0;
    int index = 0; //트럭의 인덱스
    int trucks = truck_weights.length;
    while(index <trucks || bridge_weight>0){ //truck_weights의 트럭은 다 지나고 다리에 어떤 트럭도 있어서 안된다. 
        time++;
        
        int truck_out_check = bridge_queue.poll(); //가장 앞에 있는 트럭의 값을 가져온다
        bridge_weight-=truck_out_check;// 다리 무게에서 빼버린다.
        
        if(index<trucks && bridge_weight+truck_weights[index]<=weight){
            //weight를 넘지 않는 다리 무게와 truck_weights의 인덱스를 초과하지 않을 경우에만.
            
            bridge_queue.offer(truck_weights[index]);
            bridge_weight += truck_weights[index];
            index++; // 다음 트럭으로 이동
        }else{
            //다리 무게를 초과할 경우 혹은 트럭이 다 비워졌을때는 값을 추가할 필요없고 
            // bridge_weight가 0이 될때까지 트럭을 다리 밖으로 밀어낸다.
            bridge_queue.offer(0);
        }
        
    }

# 어떤 부분에서 막혔는가?

일단 무조건 이중 포문만을 떠올렸던게 문제였다. 
**truck_weights의 트럭은 다 지나고 다리에 어떤 트럭도 있어서 안된다.**라는 조건을 지키는 반복문을 구현해야 했다. 
저 말은 곧, truck_weights의 길이보다 인덱스 값이 커지면 안되고 bridge_weight값이 0이 될때 종료된다는 조건을 가지면 된다.


    truck_out_check =bridge_queue.poll();
    if(truck_out_check!=0){ //빠져나가는 값이 0이 아니면 트럭이 나간다는 뜻이다.
        bridge_weight -= truck_out_check;
        truck_out_check =0;
    }

이 구간에서 굳이 0이 아니라는 조건을 세워서 다리의 무게를 줄일 필요가 없었다.

다리의 무게가 weight를 초과하면 안된다는 조건을 세우고 초과하지 않을 경우 다리에 트럭을 넣고 무게를 증가시키고 다음 트럭으로 이동하는 로직은 문제가 없다.

else 조건문에선 다리 무게를 초과할 경우 bridge_queue에 0을 추가하며 트럭을 밀어낸다는 의미와 truck_weights가 다 비워졌을 경우 다리의 무게가 0이 될때까지 밀어낸다는 의미가 함축되어 있었다.

아마, 풀지 못했던 이유는 어떻게 풀어야 하는가는 알고 있지만 그 로직을 구현하는데 어려움이 있었던 것 같다.
