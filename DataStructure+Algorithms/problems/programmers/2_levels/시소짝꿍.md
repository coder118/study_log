# 문제 정의
문제를 보고 바로 든 생각은 시간복잡도를 고려함.
weights의 길이가 100,000까지이기 때문에 최대 100억번 반복할 수 있기 때문이다.

1. 위에서 얘기했듯 시간복잡도 고려.
2. Map을 활용하여 중복여부를 체크하면서 문제 풀기
3. 정렬을 활용하여 뒤에오는 값들을 2/3, 1/2, 3/4만 해주면 되지 않을까, 굳이 클때 작을때 신경쓸 필요가 없다.

# 코드
    class Solution {
        public long solution(int[] weights) {
            long answer = 0;

            Arrays.sort(weights);

            Map<Double, Integer> map = new HashMap<>();

            for(int i : weights) {
                double a = i*1.0;

                double b = (i*2.0)/3.0;

                double c = (i*1.0)/2.0;

                double d = (i*3.0)/4.0;

                if(map.containsKey(a)) answer += map.get(a);
                if(map.containsKey(b)) answer += map.get(b);
                if(map.containsKey(c)) answer += map.get(c);
                if(map.containsKey(d)) answer += map.get(d);
                
                map.put((i*1.0), map.getOrDefault((i*1.0), 0)+1);
            }
            
            return answer;
        }
    }
