# 문제 정의

- 뒤에 큰 수가 있으면 그 수를 가져온다. 가장 첫번째로 찾는 값
- 없으면 -1

문제 자체는 단순했지만 가장 걸리는 부분이 시간 복잡도였다. numbers가 100만인데 이중 for문을 돌리면 10억이 되면서 10초이상의 시간이 걸리게 될 것이다.

## 이중 포문 사용

    int len = numbers.length;
    List<Integer> save = new ArrayList<>();
    for(int i =0; i<len; i++){
        boolean check  = true;
        for(int k = i+1; k<len; k++){

            if(numbers[i]<numbers[k]){
                save.add(numbers[k]);
                check = false;
                break;
            }
        }
        if(check) save.add(-1);
    }
    
    
    return save.stream().mapToInt(Integer::intValue).toArray();

첫 번째 값을 기준으로 잡고 뒤에 있는 값들을 하나씩 비교하는 이중 포문이다. save는 리스트로 필요한 값을 저장하고 int[]형태로 마지막에 바꿔주었다.
<br>
아니나 다를까, 시간 초과 문제가 발생했다.
<br>

## stack을 사용한 풀이

  int[] answer = new int[numbers.length];
        
    Stack<Integer> stack = new Stack<>();

    stack.push(0);
    for(int i = 1; i<numbers.length; i++){
        
        while(!stack.isEmpty()&& numbers[stack.peek()]<numbers[i]){
            //뒤에 큰 값이 있을 경우에 반복을 돈다.
            answer[stack.pop()] = numbers[i];
            
        }
        //현 인덱스 push
        stack.push(i);
        
    }
    while(!stack.isEmpty()){
        answer[stack.pop()] = -1;
    }

스택을 이용해서 풀면서 뒤에있는 값을 모두 반복을 도는 O(n^2)이 아니라 한번의 for문을 사용하게 만들었다.

# 배운점

스택과 인덱스의 사용을 자유롭게 할 수 있으며 반복문내에 반복문이 있다고 다 이중 포문인 것이 아니다. 조건을 달아서 전부 다 반복하지 않게 만들고 시간 복잡도를 낮출 수 있다.
