
# 문제 정의

1. 문자열에서 같은 알파벳이 2개 붙어있는 짝을 찾는다.
=> 문자열 하나씩 순회(앞 뒤 비교), substring,인덱스로 안쪼개짐

2. 찾으면 제거한다.
=> 새로운 문자열이 만들어지는 것이니까 새로운 문자열 공간이 필요
=>StringBuilder로 문자열 공간만들고 추가를 하지 않는다.

3. 다 제거 가능하면 1 반환-> 문자열이 없으면(length가0)
4. 불가능하면 0반환 -> 문자열 남아있으면(length 1이상)

<br>

**스택을 이용하면 이전 값을 저장할 수 있는 공간을 만들 수 있을것 같다.**


# 코드 구현
  
---------
        List<String> strList= Arrays.asList(s.split(""));
      
        ArrayDeque<String> before_value = new ArrayDeque<>();
        List<String> save = new ArrayList<>();
        
        for(int i = 0; i<strList.length; i++){
            
            String before = before_value.peek();
            if(before.equals(strList.get(i))){
                //이전 값과 동일한 경우 그냥 넘어가야함
                before_value.pop();
                continue;
            }else{
                save.append(before_value.pop());
                before_value.push(strList.get(i));   
            }
        }

위처럼 코드를 짜면 생기는 문제는 save에 값이 남아있다면 한번 더 반복을 해야 한다.
그리고 save를 계속 반복 확인하면서 겹치는 값이 없을때까지 반복되어야 된다는 것인데 이렇게 되면 종료 시점을 정하기 힘들어진다.
<br>
(굳이 종료시점을 정해본다면 save에 append되는 횟수와 save의 length를 구해서 동일한 경우 더이상 제거할 값이 없다는 것이니 그때 종료시킨다.)

# 해결 방법

너무 복잡하게 생각할 필요없이 stack을 이용하면 간단하게 풀 수 있다.

문자열을 stack에 채우면서 stack에 값이 있으면 들어오는 값과 비교를 한다.

    for (int i = 0; i < s.length(); i++) {
        if (stack.isEmpty()) { // 스택이 비어있다면 현재 문자를 스택에 추가
            stack.push(s.charAt(i));
        } else {
            char ch = s.charAt(i); // 현재 문자

            // 스택의 맨 위 문자와 현재 문자가 같다면 스택에서 제거
            if (stack.peek() == ch) {
                stack.pop();
            // 같지 않다면 현재 문자를 스택에 추가
            } else {
                stack.push(ch);
            }
        }
    }

중요한 점은 같지 않다면 stack에 값을 추가해서 새로 들어오는 값과 동일한지 확인하는 작업이다.
위처럼 구현하면 save를 불필요하게 반복하는 문제도 없어진다.
