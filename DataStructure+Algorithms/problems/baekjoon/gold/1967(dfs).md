[문제로](https://www.acmicpc.net/problem/1967)


# 문제 정의

1. 가장 긴 값은 가장 끝 노드를 기준으로 잡아당겼을떄 길 가능성이 높다.
2. 자식노드를 당길떄 같은 부모의 노드를 당기는 경우는 제외한다. 즉,leafnode를 당기는데 부모가 달라야 한다.
3. 몇개의 조합이 만들어질 것이고 그때 간선의 길이합을 리스트에 저장 혹은 최대힙에 저장
4. 최대값 출력

=> 잘못된 부분 수정

1. 여기에는 가중치가 들어있다. 가장 끝 값끼리 당겼을때 가장 긴값이 안 나올 수 있다. 그러므로, 루트 노드에서 가장 먼 노드를 찾는다.
2. 가장 먼 노드 위치가 루트가 되었다고 생각하고 모든 길이를 측정한다.
3. 그 중에 최대값을 구한다.

# 구현의 어려움

트리를 배열로 만들때 사실 완전 이진 트리만을 생각해서 2*i의 위치는 왼쪽 자식 노드, +1하면 오른쪽 자식 노드가 위치해야 한다고 생각했다. 하지만 트리는 항상 완전 이진트리로 구성되지 않는다.

배열에서 앞에 0은 생략했다.

    tree[child].add(new Node(parent, weight));

해당 코드는 parent만 연결을 하게 될 경우에는 트리는 아래로 내려갈 수 밖에 없다. 부모가 어떤 값인지 확인할 방법이 없다는 것이다. 여기서 위의 코드를 추가해줌으로써 자식 노드에서 부모 노드가 어떤 것인지 확인을 가능하게 해주었다.

## 주의점 (visited 사용 이유)

위 같은 경우는 1번 노드가 2번을 가르키고 2번 노드는 1번을 가르키는 **양방향 노드이다** 만약 여기서 dfs가 진행된다면 1번은 2번으로 가라하고 2번은 1번으로 가라하는 무한루프에 빠지게 될 것이다. 이 경우를 해결하기 위해 현 노드에 방문했는지 안했는지를 확인하는 로직인 visited를 만들어주는 것이다.

# 정석적인 방법

트리는 무방향성을 가지고 있다. 가중치를 가지고 가장 긴 지름을 구하기 위해서는 모든 노드가 루트 노드가 되어 dfs로 거리를 측정해서 최대 거리 값을 구하는 방식이다.

1. dfs에 들어갈 노드를 1부터 N까지 집어넣는다.
2. 양방향으로 구성되어 있기 때문에 무한 루프에 빠지지 않게 현재 자식 노드에 방문했다는 기록을 남겨둔다.
3. dfs로 탐색을 하고 leaf노드에 도달하면 방문했던 이력을 삭제해준다. (이후에 있을 탐색을 위해서)

    for (int i = 1; i <= N; i++) {
        boolean[] visited = new boolean[N + 1];
        visited[i] = true;
        dfs(tree,visited,0,i);
    }

1부터 N까지의 노드를 각각 루트로 잡고 가장 긴 길이 값을 구한다.

    public static void dfs(List<Node>[] tree, boolean[] visited, int sum,  int curNode) {

        for (int i = 0; i < tree[curNode].size(); i++) {

            Node childNode = tree[curNode].get(i);
            // 방문을 했는지 안했는지 확인
            if (!visited[childNode.nodeNum]) {
                //방문
                visited[childNode.nodeNum] =true;

                dfs(tree, visited, sum + childNode.weight, childNode.nodeNum);
                //방문이력 삭제
                visited[childNode.nodeNum] = false;

            }
            maxDiameter = Math.max(maxDiameter, sum);
        }
    }


# 최종 코드

    class Main {

        static class Node{
            private int nodeNum;
            private int weight;

            public Node(int a, int b) {
                this.nodeNum =a;
                this.weight =b;
            }
        }
        static int maxDiameter = 0;
        public static void main(String[] args) throws IOException {
            BufferedReader bf = new BufferedReader(new InputStreamReader(System.in));
            StringTokenizer st;
            int N = Integer.parseInt(bf.readLine());


            List<Node>[] tree = new ArrayList[N+1];// 트리의 노드는 0이아니라 1부터 시작되어서 +1을 함
            for (int i = 1; i <= N; i++) {
                tree[i] = new ArrayList<>(); //배열로 트리 만드는 중.
            }

            for (int i = 1; i < N; i++) {
                st = new StringTokenizer(bf.readLine());
                int parent = Integer.parseInt(st.nextToken());
                int child = Integer.parseInt(st.nextToken());
                int weight = Integer.parseInt(st.nextToken());

                tree[parent].add(new Node(child,weight));
                tree[child].add(new Node(parent, weight));
            }


            for (int i = 1; i <= N; i++) {
                boolean[] visited = new boolean[N + 1];
                visited[i] = true;
                dfs(tree,visited,0,i);
            }
            System.out.println(maxDiameter);

        }

        public static void dfs(List<Node>[] tree, boolean[] visited, int sum,  int curNode) {

            for (int i = 0; i < tree[curNode].size(); i++) {

                Node childNode = tree[curNode].get(i);
                // 방문을 했는지 안했는지 확인
                if (!visited[childNode.nodeNum]) {
                    //방문
                    visited[childNode.nodeNum] =true;

                    dfs(tree, visited, sum + childNode.weight, childNode.nodeNum);
                    //방문이력 삭제
                    visited[childNode.nodeNum] = false;

                }
                maxDiameter = Math.max(maxDiameter, sum);
            }
        }
    }

# 배운점

1. 양방향으로 트리가 이어져 있을 경우 무한루프에 빠질 수 도있으니 부모 노드에 방문이 되어있는 상태라고 visited에 기록해줘야 하는 것.
2. 완전 이진 트리가 아닌 불완전한 트리를 구현하는 방법
3. 2번에서 구현할때 배열의 인덱스가 현재 위치 노드가 되고 그 값이 자식 노드가 되게 하는 방법.


# 수정 사항.

위의 방법으로도 구하는 것이 가능하지만, 트리의 모든 노드를 루트 노드로 선택해서 가중치를 더해가며 최대 지름을 찾는 방식이다. 
<br>
확실하다는 장점이 있지만 노드의 수만큼 dfs가 반복되어야 한다는 큰 문제가 있다. 실제로 백준 문제를 풀이할때 걸린 시간이 3056ms가 걸렸다. 

## 해결책

1. 루트 노드에서 가장 먼 노드를 탐색한다. (문제의 루트 노드는 1이다.)
2. 가장 먼 노드에서 다시 가장 먼 노드를 탐색한다.(최대의 지름 길이가 된다.)

            max = Math.max(sum, max);

어제와 다르게 tree가 존재하지 않고 visited도 따로 넣어주지 않는다. **static으로 전역적으로** 사용할 수 있게 해두었다.

설명을 하자면 for문은 어제 for문과 동일하다. 어제는 현 노드 내부에 있는 자식 노드가 몇개인지 파악하고 접근했다면 오늘은 자식 노드로 바로 접근할 수 있게 for문을 수정해준 것이다.

    visited = new boolean[N + 1];

## 결론

어제 진행한 풀이는 모든 노드를 다 뒤져서 최대로 긴값을 찾는 과정이었다면, 오늘은 루트 노드에서 가장 먼 노드를 찾고 그 노드에서 가장 먼 노드를 찾으면 최대값이 나온다는 성질을 이용해서 단 2번의 dfs 과정을 통해 찾을 수 있었다.
