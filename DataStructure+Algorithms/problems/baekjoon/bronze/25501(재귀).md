# 문제 정의

문자열을 입력했을 때, 앞에서 부터 읽어도 뒤에서 부터 읽어도 동일한 문자열이 나와야 한다.

1. 문자열을 인덱스로 접근이 가능하다. charAt을 사용하면 하나 단위로 쪼개서 인덱스로 접근이 가능
2. 첫 인덱스와 끝 인덱스를넣어서 첫 자리와 끝자리를 비교한다.
3. 동일한 값이 없으면 0을 반환
4. 첫 인덱스 값은 증가할 것이고 끝 인덱스값은 감소하다 보면 중간에 값이 같아지거나 첫 인덱스 값이 커지는 경우가 있다. 이럴 경우에는 팰린드롬이라는 것이 되므로 1을 반환한다.

## 코드 구현


    public class Prac1 {
        private static int check =0;

        public static void main(String[] args) throws IOException {
            BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

            int N = Integer.parseInt(br.readLine());

            for (int i = 0; i < N; i++) {
                String str = br.readLine();

                int OX = isPalindrome(str);
                System.out.printf("%d %d%n", OX,check);
                check=0;
            }

        }

        private static int isPalindrome(String str) {

            //문자열의 처음과 끝을 하나씩 비교한다.
            //첫 인덱스 값이 끝 인덱스 값보다 커지는 경우는 팰린드롬이라는 것.
            //인덱스를 통해 비교하는 과정에서 문자열이 동일하지 않으면 팰린드롬이 아니다.
            return recursion(str,0,str.length()-1);

        }

        private static int recursion(String str, int first, int last) {
            check+=1;
            if(first>= last) return 1;
            else if(str.charAt(first)!=str.charAt(last)) return 0;

            return recursion(str, first + 1, last - 1);
        }

    }

## 주의할 점

    if(first>= last) return 1;

이 부분에서 =을 빼먹었다. 첫번째 값이 커질 뿐만 아니라 값이 동일할 경우도 계산에 넣어야 한다. 사소한 것을 빼먹으면 문제가 크게 발생할 수 있으므로 주의해야 한다.
