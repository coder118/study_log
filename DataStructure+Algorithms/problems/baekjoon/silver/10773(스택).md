# 문제 정의

재현 0을 외치면 재민이 적은 가장 최신 숫자를 지운다.
재민은 그 숫자를 전부 적고 합을 알고 싶어한다.
<br>
숫자의 전체 합을 구하기 위해선 숫자를 담을 바구니가 필요하다. 가장 최근에 들어가는 값을 손쉽게 제거할 수 있는 스택을 사용했다.

# 코드 구현

    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

    int N = Integer.parseInt(br.readLine());

    Stack<Integer> stack = new Stack<>();

    for (int i = 0; i < N; i++) {
        int num = Integer.parseInt(br.readLine());
        if(num==0){
            //0이 입력되면 가장 최근 값을 뺀다.
            //그리고 push하지 않는다.
            stack.pop();
            continue;
        }
        stack.push(num);
    }

    int sum = 0;
    while (!stack.isEmpty()) {
        int num = stack.pop();
        sum+=num;
    }

    System.out.println(sum);

## 실수했던 점

처음에 stack에 값을 다 저장하고 0이 나오면 최근의 값을 제거하게 구현을 했다가 스택에 값을 미리 다 저장해두면 0이 나올때마다 가장 최근에 입력한 값을 뺼 수 가 없다는 것을 알고 사용자가 입력을 하는 시점에 숫자를 제거해야 한다는 것을 알고 수정했다.

## 간단한 stack 클래스 구현해서 풀어보기


    class Stack2 {
        private int[] arr;
        private int idx,length;

        public Stack2(int N) {
            this.arr = new int[N];
            this.idx =-1;
            this.length = 0;
        }

        public void push(int num) {
            arr[++idx] = num;
            this.length +=1;
        }

        public int pop() {
            this.length-=1;
            return arr[idx--];
        }

        public boolean isEmpty() {
            return this.length == 0;
        }

    }   

정말 간단하게 예제를 통과할 수 있을 정도로만 구현했다. 구현되어있는 라이브러리가 아닌 직접 구현을 해보면서 메모리의 동작이 어떻게 진행되는지 생각할 수 있었다.

<br>
실행코드는 거의 동일하지만 Stack 클래스를 호출하는 부분만 바꿔주면된다.

    Stack2 stack = new Stack2(N);
