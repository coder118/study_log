# 문제 정의

스택을 구현하는 문제인데 다른 점은 숫자로 push나 pop 명령을 실행하게 구현해야 하며, push할때는 공백 처리후 입력값이 들어가야 한다는 점을 해결해야 했다.

1. 숫자로 분기하는 switch문을 만든다.
2. push(=1)이 입력받을 경우 공백 뒤에 입력값이 들어올때 StringTokenizer을 이용해서 구분해준다.

    public class Prac1 {

        public static void main(String[] args) throws IOException {
            BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

            int N = Integer.parseInt(br.readLine());

            Stack2 stack = new Stack2(N);
            for (int i = 0; i < N; i++) {
                StringTokenizer st = new StringTokenizer(br.readLine());
                int input_value=0;
                int num = Integer.parseInt(st.nextToken());
                if (st.hasMoreTokens()) {
                    input_value= Integer.parseInt(st.nextToken());
                }

                switch(num) {
                    case 1:
                        stack.push(input_value);
                        break;
                    case 2:
                        System.out.println(stack.pop());
                        break;
                    case 3:
                        System.out.println(stack.size());
                        break;
                    case 4:
                        System.out.println(stack.isEmpty());
                        break;
                    case 5:
                        System.out.println(stack.peek());
                        break;

                }

            }

        }


    }

    class Stack2 {
        private final int[] arr;
        private int idx,length;

        public Stack2(int N) {
            this.arr = new int[N];
            this.idx =-1;
            this.length = 0;
        }

        public void push(int num) {
            arr[++idx] = num;
            this.length +=1;
        }

        public int pop() {
            if (this.length == 0) {
                return -1;
            }
            this.length-=1;
            return arr[idx--];
        }

        public int isEmpty() {
            return this.length == 0 ? 1 : 0;
        }

        public int peek() {
            if (this.length != 0) {
                return arr[idx];
            }
            return -1;
        }

        public int size() {
            return this.length;
        }
    }

## 문제점

동작하는 것에는 문제가 없이 잘 동작했다. 하지만 큰 문제가 하나 있었는데 System.out.println이 반복문 내에서 반복되는 문제가 생겼다. N이 100만이라면 100만 번의 시스템 I/O가 발생하게 되어서 속도적인 부분에서 문제가 생길 것이다.
